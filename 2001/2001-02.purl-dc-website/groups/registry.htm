Working Group Charter:

The DCMI Registry Working Group aims to establish a metadata registry to support the activity of the DCMI. Exploration of the role of a DCMI registry is becoming a priority task in order to manage Dublin Core elements and qualifiers as they are used and adapted for an increasingly diverse set of applications.

The registry is intended to fulfill various purposes such as

  • To manage DCMI namespace(s)
  • To provide an authoritative source of information regarding DCMI for
    humans and software
  • To facilitate take up of DCMI
  • To manage change and evolution of the DCMI vocabulary

The WG will identify requirements for a DCMI registry, scope the registry, recommend policy and process and implement a solution. The intention is to have a phased delivery, starting with basic functionality
and content.

Background:

The aim is to produce a Phase 1 implementation by end of January 2000.

Production of these deliverables will be an iterative process, there has already been considerable background work within the DCMI community, and a prototype is available at http://eor.dublincore.org

The DCMI working group will act as a focus for activity regarding metadata registries within the DCMI. It will disseminate information about its work to dc-general and dc-ac. The DCMI Registry working group will collaborate with:

  • the Interest Group for Dublin Core in Multiple Languages, which maintains an active interest in issues related to the registry
  • the DCMI Architecture WG

The Registry WG will liaise as appropriate with registry activity outside the DCMI. In particular with:

  • ISO 11179 community
  • W3C RDF Schema Group
  • W3C XML Schema Group
