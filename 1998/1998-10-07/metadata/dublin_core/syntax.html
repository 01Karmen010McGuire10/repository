HTML 4.0

In HTML 4.0 The META element  the following attributes, the permitted values and their interpretation are profile dependent:

  • NAME: This attribute specifies a property name.
  • CONTENT: This attribute specifies a property's value.
  • SCHEME: This attribute names a scheme to be used to interpret the property's value.
  • HTTP-EQUIV: This attribute may be used in place of the name attribute. HTTP servers use this attribute to gather information for HTTP response message headers.
  • LANG: language tagging based on RFC 1766, Tags for the Identification of Languages (http://ds.internic.net/rfc/rfc1766.txt)

The META element can be used to describe properties of a document (e.g., author, expiration date, a list of key words, etc.) and assign values to those properties. This specification does not define a normative set of properties.

The additional attributes SCHEME and LANG are useful for qualifying content. For example:

<META name="DC.creator" lang="fr" content="Arnaud Le Hors">
<META name="DC.subject" scheme = "DDC" content="541.34">

RDF Resource Description Framework

RDF--the Resource Description Framework--is a framework for metadata; it provides interoperability between applications that exchange machine-understandable information on the Web. RDF emphasizes facilities to enable automated processing of Web resources. RDF metadata can be used in a variety of application areas including resource description as defined by the Dublin Core.

The RDF work is under the auspice of the World Wide Web Consortium (W3C), the organization facilitating standards for the Web. RDF started as an extension of the W3C PICS content description technology. It is now also drawing upon the XML design as well as technology submissions by Microsoft (XML Web Collections) and Netscape (XML/MCF). Other documents, such as Microsoft's XML-Data paper, Site Map proposals, and the Dublin Core/Warwick Framework have also influenced the RDF design.

There is currently (Wed Sep 24 16:41:04 EDT 1997) a core group of people from the Dublin Core community including Ron Daniel, Renato Iannella and Eric Miller that are working on representing the Dublin Core in RDF. We are targeting the DC5 workshop for the initial presentation of this work.

Additional information on RDF and W3C's metadata initiative are availiable from their corresponding web pages.
