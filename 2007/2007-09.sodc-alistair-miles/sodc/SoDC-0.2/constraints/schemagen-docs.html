1. Introduction

@@TODO

The SoDC-CL language is designed to enable the generation of concrete, application-specific syntax validation tools from SoDC-CL documents.

Because SoDC-CL is defining constraints over the abstract graph syntax, it is of course possible to generate validators from a SoDC-CL document for a range of concrete syntaxes, using a variety of technologies.

2. Stylesheet Transformation

The SchemaGen stylesheet is a specific implementation of SoDC-CL, which generates a Schematron schema from a SoDC-CL document. The Schematron schema can then be used to validate specific SoDC-XML documents with respect to an "application profile". The SchemaGen stylesheet is at the link below:

  • schemagen.xsl -- SchemaGen Schema Generation Stylesheet

For example, the SoDC-CL document in the example above was used to generate a Schematron schema, which can then be used to validate an SoDC-XML document using a Schematron implementation.
