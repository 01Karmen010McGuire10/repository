2. Architecture

The starting point for the SoDC information architecture is the Resource Description Framework (RDF) abstract syntax. The RDF abstract syntax provides a foundation for graph-based (meta)data -- data that describes things which are related to other things.

Informally, the RDF abstract syntax can be summarised as follows. A graph describes resources, each resource has properties which have values. The value of a property can be a URI, a literal or a blank node (a.k.a. "anonymous" resource). A literal can be a plain literal or a typed literal. A resource can be identified by a URI, or can be "anonymous". A property is identified by a URI.

So, for example, a graph might describe a book titled "Winnie the Pooh", created by a person called "A. A. Milne".

3. Design Goals

SoDC-XML is a concrete XML syntax for graph-based (meta)data. SoDC-XML has the following design goals:

  • Concrete encoding of the RDF abstract syntax;
  • Suitable for embedding in (meta)data harvesting protocols, e.g. OAI-PMH;
  • Constrained by a W3C XML schema, to support basic syntax validation;
  • Can be constrained by an application-specific schema, to support higher levels of syntax validation.

4. Syntax Specification

SoDC-XML is a concrete XML syntax for graph-based (meta)data, built on the RDF abstract syntax. The syntax is defined by a W3C XML Schema, at the following location:

  • sodc-xml.xsd -- SoDC-XML Schema

4.1. Examples

A graph, describing a book titled "Winnie the Pooh"...

<?xml version="1.0" encoding="UTF-8"?>
<graph xmlns="http://purl.org/net/sodc/xml">

  <resource>
    <uri>http://example.com/book#1</uri>
    <property uri="http://purl.org/dc/elements/1.1/title">
      <value>
        <plainliteral lang="en">Winnie the Pooh</plainliteral>
      </value>
    </property>
  </resource>

</graph>

A graph, describing a book created by a person...

<?xml version="1.0" encoding="UTF-8"?>
<graph xmlns="http://purl.org/net/sodc/xml">

  <resource>
    <uri>http://example.com/book#1</uri>
    <property uri="http://purl.org/dc/elements/1.1/creator">
      <value>
        <uri>http://example.com/person#1</uri>
      </value>
    </property>
  </resource>

</graph>

A graph, describing the price of a book...

<?xml version="1.0" encoding="UTF-8"?>
<graph xmlns="http://purl.org/net/sodc/xml">
    
  <resource>
    <uri>http://example.com/book#1</uri>
    <property uri="http://example.com/property#price">
      <value>
        <typedliteral datatype="http://www.w3.org/2001/XMLSchema#decimal">9.99</typedliteral>
      </value>
    </property>
  </resource>
  
</graph>

A graph, describing a book titled "Winnie the Pooh", costing 9.99, created by a person called "A. A. Milne"...

<?xml version="1.0" encoding="UTF-8"?>
<graph xmlns="http://purl.org/net/sodc/xml">  

  <resource>
    <uri>http://example.com/book#1</uri>
    <property uri="http://purl.org/dc/elements/1.1/title">
      <value>
        <plainliteral lang="en">Winnie the Pooh</plainliteral>
      </value>
    </property>
    <property uri="http://purl.org/dc/elements/1.1/creator">
      <value>
        <uri>http://example.com/person#1</uri>
      </value>
    </property>
    <property uri="http://example.com/property#price">
      <value>
        <typedliteral datatype="http://www.w3.org/2001/XMLSchema#decimal">9.99</typedliteral>
      </value>
    </property>
  </resource>

  <resource>
    <uri>http://example.com/person#1</uri>
    <property uri="http://xmlns.com/foaf/0.1/name">
      <value>
        <plainliteral lang="en">A. A. Milne</plainliteral>
      </value>
    </property>
  </resource>
  
</graph>
