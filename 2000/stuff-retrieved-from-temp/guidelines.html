Alternative link to graphical or text-only version of document.

[Priority 1] A Web content developer must satisfy this checkpoint.

IMAGES:Provide a text equivalent for every non-text element using ALT and LONGDESC.

Code example:

<IMG SRC="/images/placebo.gif" ALT="Brief description of what the image is." LONGDESC="Specifies a URL of a document that contains a longer description of the element that what the content of the ALT or TITLE attributes reveal." WIDTH="32" HEIGHT="32" BORDER="0">

LANG:Use the LANG attribute to clearly identify changes in the natural language of a document's text and any text equivalents.

Code examples:

<STRONG LANG="de">German</STRONG>

Display examples:

German

HEADER ELEMENTS:Use header elements to convey document structure and use them according to specification.

Code examples:

<H1>Header 1</H1>
<H2>Header 2, subsection of Header 1</H2>
<H3>Header 3, subsection of Header 2</H3>
<H4>Header 4, subsection of Header 3</H4>

Header 1

BLOCKQUOTE:Intended to set off a long quote inside a document.
CITE:A URL pointing to an online source document from which the quotation is taken.

Code examples:

<BLOCKQUOTE CITE="http://www.macromedia.com/">What the Web can be.</BLOCKQUOTE>

Display examples:

What the Web can be.

[Priority 3] A Web content developer may address this checkpoint.

ABBR:Provides an encapsulation and enumeration mechanism for abbreviations that appear in the body text.

Code examples:

<ABBR TITLE="Ohio">OH</ABBR>
<ABBR LANG="de" TITLE="und so weiter">usw.</ABBR>

Display examples:

OH
usw.

ACRONOYM:Provides an encapsulation and enumeration mechanism for acronyms that appear in the body text.

Code examples:

<ACRONYM TITLE="Dublin Core Metadata Initiative">DCMI</ACRONYM>

Display examples:

DCMI
