Accessibility Guidelines

The DCMI Web site supports Priority 1 checkpoints for accessibility as per the W3C recommendation for Web Content Accessibility Guidelines 1.0. The Priority 1 checkpoint is as follows:

Priority 1
A Web content developer must satisfy this checkpoint. Otherwise, one or more groups will find it impossible to access information in the document. Satisfying this checkpoint is a basic requirement for some groups to be able to use Web documents.

The following are the applied guidelines and their corresponding Priority 1 checkpoints:

Guideline 1. Provide equivalent alternatives to auditory and visual content.

Checkpoint Currently Implemented?

Guideline 2. Don't rely on color alone.

Checkpoint Currently Implemented?
Ensure that all information conveyed with color is also available without color, for example from context or markup. Yes

Guideline 4. Clarify natural language usage.

Checkpoint Currently Implemented?
Clearly identify changes in the natural language of a document's text and any text equivalents (e.g., captions). Yes

Guideline 5. Create tables that transform gracefully.

Checkpoint Currently Implemented?
