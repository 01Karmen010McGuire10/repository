Directory Structure

All content should be kept in subdirectories, named according to the list below. A general principle to follow is to group similar items into their own directories. Directory names are always lowercase unless noted.

/css/

The css directory contains all external Cascading Style Sheet files. These files should not be indexed.

/images/

The images directory contains all GIF, JPEG, and PNG files; in short, any images for the website. Subdirectories within the images directory should be created for similarly used images, such as for the header, logos, news items, and templates. These files should not be indexed.

/js/

The js directory contains all external JavaScript files. These files should not be indexed.

The current JavaScript file (default.js) is linked from every page in the site. The function of default.js is to refresh the browser window if it is resized in Netscape. It also sets up browser detection variables. These are not currently used for anything, but were added for the possibility of writing a specific Cascading Style Sheet for each browser and platform.

/Library/

Libraries are Macromedia Dreamweaver's way to store page elements such as images, text, and other objects that can be reused or updated frequently throughout a website. These elements are called library items.

The Library directory contains library item files created by Dreamweaver. These are nonpublic files that should not be indexed or accessible by the general public. (Note: Dreamweaver capitalizes this directory name automatically. It should NOT be changed to lowercase.)

/media/

The media directory contains any type of audio, video, Shockwave, Flash, or other multimedia file type. Descriptive subdirectories within the media directory should be created to adequately explain the file type (i.e. swf for Flash movies). These files should not be indexed.

/pdf/

The pdf directory contains Adobe Portable Document Format files.

/psd/

The psd directory contains graphic design files created with Adobe Photoshop. These are nonpublic files that should not be indexed or accessible by the general public.

/templates/

The templates directory contains HTML files used by content creators. These files that should not be indexed and should only be accessible to a select audience.
