Linking URLs

Absolute paths

Absolute paths provide the complete URL of the linked document, including the protocol to use (usually http:// for web pages). For example, http://www.w3c.org/ is an absolute path. Absolute paths are only used to link to a document on another server.

Root-relative paths

Root-relative paths provide the path from the site's root folder to a document. Root-relative paths should always be used for links between documents within the DCMI website.

A root-relative path begins with a leading forward slash, which stands for the site root level. For example, /about/organization/committemembers.shtml is a root-relative path to the committemembers.shtml file in the organization subfolder of the about subfolder of the site's root level.

Links to the default page in a directory, index.shtml, should not include the default page in the link. For example, /about/contact/ is the proper path to the index.shtml file in the contact subfolder of the about subfolder of the site's root level.

A root-relative path often provides the best way to specify links in a website in which you need to frequently move HTML files from one folder to another. When you move a document that contains root-relative links, you don't need to change the links; for example, if your HTML files use root-relative links for dependent files (such as images), then if you move an HTML file, its dependent-file links are still valid. However, when you move or rename the documents linked to with root-relative links, you do need to update those links, even if the documents' paths relative to each other haven't changed. For example, if you move a folder, all root-relative links to files within that folder must be updated.

Root-relative links are interpreted by servers, not by browsers, so if you open a local page that uses root-relative links in your browser, the links will not work. Previewing pages in framesets that use root-relative links results in similar problems.
