Adding Documents to the DCMI Web Site

  1. Create a directory.

    Directories are created and named based on the date that the document is created/issued. All documents belong in the /documents/ directory. The appropriate subdirectory is based on the year/month/day/name of document/

    Examples:

    /documents/2000/07/14/dcmes-xml/

    /documents/1999/07/02/dces/

    NOTE: Name of document should be fairly generic. The intent is to maintain this name through all the stages a document can pass through (note/working draft/proposed recommendation/recommendation).

  2. Once the directory is created and correctly added onto the DCMI web site, you can go ahead and create/move the files into the directory. There should be one file that is titled "index.shtml"
  3. All links to other files within the DCMI site should be relative.
  4. Once the file is ready to go, you will need to update the following files:
    • The documents list on the documents index page: /documents/index.shtml
    • Create a softlink (for a first time document) or update the softlink to the most recent version of the document.
      • in the /documents/ directory create a new directory based on the file name (example: /documents/filename/)
      • copy a softlink file into the new directory.
      • update the link to the latest copy of the document. (example: /documents/2000/10/05/education-namespace/)
