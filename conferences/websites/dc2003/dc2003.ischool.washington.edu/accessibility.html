Chair: Liddy Nevile, Liddy.Nevile@motile.net

There are two working sessions for those interested in standardizing the specifications for metadata relating to accessibility. Please note that "accessibility" in this context is about making all resources and services accessible to everyone regardless of the device providing access or the individual's disability.

The DCMI Accessibility Interest Group has considered "compliance" and "functional" metadata as being slightly different. People who have developed metadata elements or qualifications that fit either category are invited to submit a short proposal for a brief presentation in one of the two sessions at the DC 2003 Conference.

Session 1: Compliance Metadata
Wednesday, 1 October 2003, 8:30-10:00 Cove Room

This session will consider the use of DCMES elements and recommended extensions to form an application profile that can be used to discover resources that comply with a range of sets of requirements and regulations, eg WCAG 1.0, s.508, etc.

Session 2: Functional Metadata
Wednesday, 1 October 2003, 15:30-17:30 Cove Room

This session will consider, among other work, the recently released user profile metadata specification (see ACCLIP at http://www.imsproject.org/), the developing Alternative Interface Access Protocol (see http://www.incits.org/tc_home/v2.htm).

We will discuss opportunities for a DC working group to participate in a collaborative effort with other organisations to develop an application profile for a new discovery element (dc:accessibility) that matches resources and devices to user profiles.

Logistics:

Please send proposals to Liddy Nevile as soon as possible. Proposals from people not attending the conference will be welcomed, and we will endeavour to report on them. Both sessions will be documented and the outcomes posted to the DCMI Accessibility Interest Group Web site.
