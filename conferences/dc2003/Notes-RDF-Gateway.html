Faceted Search:

Note that the search engine for the Gateway+ is very different from the previous GEM search engines. While the past GEM search engines may be characterized as basic boolean engines that permited the construction of fairly complex queries, they may also be characterized as relatively difficult to use for novice searchers. The Gateway+ search engine has been designed to eliminate boolean complexity and still allow the end user to use it in sophisticated ways.

The plus (+) in Gateway+ stands for what is called faceted searching. Faceted searching is a mechanism for combining the attributes of searching and browsing into a single process. A Gateway+ search begins like any other simple Web search engine search–i.e., a Google®-like search– by entering a brief text string that describes the information need.

Generally, the results of the Gateway+ text query is a fairly large (sometimes very large) retrieval set. When a user inputs a simple text query such as "earth fire water air", the terms are executed with a boolean "or". At this time (and with the exceptions noted in the table below), boolean operators are not supported in the free text query box. When the user adds selections from the GEM subject drop-down menus or from the grade level selection buttons, those facets of the search are combined using the boolean "and".

Usually, but not always, the second phase of the faceted search is to begin narrowing down the retrieval set based on various facets of the resources in that set. For example, one facet of the records in the retrieval set is their "Resource Type"–i.e., some may be lesson plans, others are activities, and still others may be applets and computer applications.

At the top of each retrieval set returned by Gateway+ is a set of facets and clickable terms under each facet that have been derived from the retrieval set. So, for example, an end user that is only interested in looking at lesson plans can reduce the set of retrieved resources to only lesson plans by clicking on the term in the facet list for Resource Type. In like fashion, picking the term Parents from the Audience element will reduce the retrieval set even further to only those resources that are lesson plans and useful for parents. This form of browsing–burrowing down into the retrieval set–may be continued until the retrieval set contains only those resources that fit the desired facet choices of interest.

When retrieval sets are displayed to the user, the sequence of records in the display (i.e., relevance ranking) is a function of weighting the terms in the query based on the elements in which they occur. Thus, query terms are given weight in the descending order of the following elements: keywords, subject, title, description, anywhere else.

Search Operators:

Operator
Description
+

A leading plus sign indicates that this word must be present in every element returned.

Example:
+apple +juice

Find elements that contain both "apple" and "juice."

Example:
+apple macintosh

Find elements that contain at least the word "apple'', but rank it higher if it also contain "macintosh''.
-

A leading minus sign indicates that this word must not be present in any element returned.

Example:
+apple -macintosh

Find elements that contain at least the word "apple'' but not "macintosh''.
< >
( )

These two operators are used to change a word's contribution to the relevance value that is assigned to an element. The < operator decreases the contribution and the > operator increases it. Parentheses are used to group words into subexpressions.

Example:
+apple +(>pie <strudel)

Find elements that contain at least the word "apple'' and "pie'', or "apple'' and "strudel'' (in any order), but rank "apple pie'' higher than "apple strudel''.
~

A leading tilde acts as a negation operator, causing the word's contribution to the element relevance to be negative. It's useful for marking noise words. An element that contains such a word will be rated lower than others, but will not be excluded altogether, as it would be with the - operator.
*

An asterisk is the truncation operator. Unlike the other operators, it should be appended to the word, not prepended.

Example:
apple*

Find elements that contain at least the word "apple'' and "applet'', or "appleton''.
" "

The phrase that is enclosed in double quotes matches only elements that contain this phrase literally as it was typed. Note: At this time, the exact text match ignores words of three letters or less.

Known Bugs:

The following table contains known bugs and identified feature issues. We are working our way through the bugs. Substantial feature requests will be rolled over into the EG implementation of the Gateway+.

SF#
Description
 
Missing elements in the full record display: Currently, the full record display is incomplete. Missing are a number of elements as well as linked values (e.g., relation URLs that are not active links). We will continue cleaning up these issues over the next week.
 
Screen formatting of GEM subjects in full display is incorrect. Currently, GEM subjects at both the first and second level appear in the full display but not in their hierarchical form. We will fix this display problem over the next week.
 
Grade selections on the search screen. Currently, the user cannot execute a search consisting only of a grade or grade range selection. In other words, the user cannot ask: "Give me everything you have for the third grade." Grade selections as qualifiers of either a full text entry or a GEM subject selection works.
 
GEM subject terms in the faceted list. We are displaying GEM vocabulary terms (both first and second level) as a simple list in the facets. This is not an error but a design choice.
 
Facet terms displayed is incomplete. Given the limited screen real estate for the facet display, only a limited number of terms are displayed unto each facet. We plan to have a "[more ...]" button at the bottom of all facet lists where additional terms (beyond the limit set for the default display) exist. By clicking on "[more ...]", the user can get a list of the remaining terms for that facet.
 
Rights > Cost display. Unlike the Inktomi version of the Gateway, Gateway+ has removed the end user free/fee-based selection from the query page and will permit the user to "toggle" back and forth using facet terms. Note: The Rights facet (6th on the facet display) has not been implemented. We will be adding it to the display within a week or so.
